### Sortie 1: Public canonical notes-stripper (`GlosaInlineNotes`) — FR1

**Work Unit**: GlosaCore-API · **Layer**: 0

**Priority**: 20 — keystone. Transitively blocks all 5 downstream sorties (highest dependency depth); establishes the `GlosaInlineNotes` API reused by Sortie 2 and Sortie 3. Must execute first.

**Entry criteria**:
- [ ] First sortie — no prerequisites.

**Tasks**:
1. Add `public enum GlosaInlineNotes` in a new file
   `Sources/GlosaCore/GlosaInlineNotes.swift` with
   `public static func strip(_ dialogue: String) -> String` and
   `public static func split(_ dialogue: String) -> (stripped: String, notes: [String])`.
   It is the single source of truth for `[[ … ]]` inline-note stripping.
2. Refactor `GlosaParser.extractInlineNotes`
   (`Sources/GlosaCore/GlosaParser.swift:491`) so the prose it strips is produced
   by — or proven byte-identical to — `GlosaInlineNotes.split`. The parser still
   needs per-marker offsets/attributes, so route the *stripping/gap* logic through
   the shared API rather than duplicating the regex; the canonical stripped buffer
   must remain identical to today's output.
3. Document in the API doc-comment that offsets are `unicodeScalars.count` indices
   into the `stripped` string.
4. Add a `GlosaCoreTests` test proving `GlosaInlineNotes.split(raw).stripped`
   equals the parser's stripped text for the same input across at least: plain
   prose, single `[[<breath/>]]`, single `[[<pause/>]]`, interleaved breath+pause,
   and an emoji/combining-mark fixture (AC1).

**Exit criteria**:
- [ ] `Sources/GlosaCore/GlosaInlineNotes.swift` exists exporting `strip` and `split`.
- [ ] `xcodebuild build -scheme glosa-av-Package -destination 'platform=macOS'` succeeds.

      including the new parser-vs-`split` equivalence test (AC1).
- [ ] `grep -n "replacingOccurrences" Sources/GlosaCore/GlosaParser.swift` shows the
      inline-note stripping regex is no longer duplicated in the parser (routes

### Sortie 2: Compile-to-DTO entry point (`GlosaLineAnnotation` + `compileAnnotations`) — FR2

**Work Unit**: GlosaCore-API · **Layer**: 1

**Priority**: 5.75 — lower dependency depth (blocks only Sortie 6), but high risk (offset indexing across emoji/combining marks). Depends only on Sortie 1, so it is **parallelizable** with the Sortie 3 → 4 → 5 chain (see § Parallelism Structure). Kept adjacent to Sortie 1 for GlosaCore-API work-unit coherence.

**Entry criteria**:
- [ ] Sortie 1 complete (`GlosaInlineNotes.split` available; build/tests green).

**Tasks**:
1. Add `public struct GlosaLineAnnotation: Codable, Sendable` and
   `public struct PausePointDTO: Codable, Sendable` in a new file
   `Sources/GlosaCore/GlosaLineAnnotation.swift` with the exact fields from the
   requirements (`spokenText`, `breathOffsets: [Int]`, `breathStrengths: [String]`,
   `instruct: String?`, `pausePoints: [PausePointDTO]`; `PausePointDTO` =
   `offset`, `lengthMs`, `named`).
2. Add `public func compileAnnotations(fountainNotes: [String], rawDialogueLines: [(character: String, rawText: String)]) throws -> [Int: GlosaLineAnnotation]`
   that strips each `rawText` via `GlosaInlineNotes.strip` (consumer never strips),
   calls the existing `compile(fountainNotes:dialogueLines:)`, and projects
   `CompilationResult` into the DTO: `spokenText` = stripped line; `breathOffsets` /
   `breathStrengths` from `breathPoints[i]` (strength via `.rawValue`); `instruct`
   from `instructs[i]`; `pausePoints` from `pausePoints[i]`. **`PauseLength` →
   `PausePointDTO` mapping (per resolved OQ-3)**: named presets →
   `lengthMs` = {`comma` 150, `semicolon` 250, `period` 400, `emDash` 600,
   `beat` 1000} and `named` = {`"comma"`, `"semicolon"`, `"period"`, `"em-dash"`,
   `"beat"`}; `.explicit(seconds)` → `lengthMs = Int((seconds * 1000).rounded())`
   and `named = nil`. Centralize these constants in one place (single source of
   truth) so they stay aligned with future SwiftVoxAlta calibration.
3. Leave the existing `compile` / `CompilationResult` API **unchanged** (back-compat).
4. Confirm/annotate that every public type crossing the boundary
   (`GlosaLineAnnotation`, `PausePointDTO`, and any retained
   `BreathStrength`/`PauseLength`/`GlosaDiagnostic`/`InstructProvenance`) is
   `Sendable` and (where serialized) `Codable`.
5. Add `GlosaCoreTests` proving: (a) offsets in `breathOffsets` index correctly
   into `spokenText` for a multi-breath fixture incl. emoji / combining marks /
   `after=` positioning; (b) splitting `spokenText.unicodeScalars` at
   `breathOffsets` round-trips losslessly (AC2); (c) a `PausePointDTO` mapping test
   asserting each named preset → its `lengthMs`/`named` pair and an `.explicit`
   value → rounded `lengthMs` with `named = nil`.

**Exit criteria**:
- [ ] `Sources/GlosaCore/GlosaLineAnnotation.swift` exists exporting the DTOs and `compileAnnotations`.
- [ ] `xcodebuild build -scheme glosa-av-Package -destination 'platform=macOS'` succeeds.
- [ ] `xcodebuild test -scheme glosa-av-Package -destination 'platform=macOS'` passes,
      including the offset-indexing, round-trip, and pause-mapping tests (AC2 + OQ-3).
- [ ] The existing `compile(fountainNotes:dialogueLines:)` signature and
      `CompilationResult` are unchanged (`git diff` shows additions only in those files).

## Parallelism Structure

**Critical Path**: Sortie 1 → Sortie 3 → Sortie 4 → Sortie 5 → Sortie 6 (length: **5 sorties**).

**Parallel Execution Groups**:
- **Group 1** (after Sortie 1 completes — two independent branches, both depend only on S1):
  - Branch A (critical path): Sortie 3 → Sortie 4 → Sortie 5
  - Branch B: Sortie 2 (FR2 DTO; touches only `GlosaCore`, untouched by the move)
- **Group 2** (join point): Sortie 6 — requires Branch A (S4, S5) **and** Branch B (S2).

**Agent Constraints** — **CRITICAL**: sub-agents do NOT run builds; only the supervising agent builds.
- **Every one of the 6 sorties terminates in a build/test gate** (`xcodebuild` or MCP `swift_package_*`). Therefore all 6 are **SUPERVISING AGENT ONLY** and cannot be fully delegated to parallel sub-agents.
- **Realistic parallelism is low**: the maximum concurrency is the 2 independent branches above, but because each branch's sorties build, the supervising agent must serialize the build gates. The useful sub-agent work here is non-build preparation only — e.g. drafting `../glosa-tools/Package.swift` text (S3), composing the `CHANGELOG.md` entry (S6), or authoring test fixtures — with the supervising agent owning every compile/test step.
- **Recommended dispatch order** (single supervising agent, dependency-respecting, priority-front-loaded): S1 → S3 → S4 → S2 → S5 → S6. S2 may be pulled earlier (right after S1) if the operator prefers to close out GlosaCore-API before touching the move; it does not sit on the critical path either way.

**Missed Opportunities**: None exploitable for true concurrency — the universal build-gate constraint collapses the two logical branches onto one supervising agent. Parallelism is documented for clarity, not for multi-agent fan-out.
